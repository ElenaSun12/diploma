# План тестирования и автоматизации Android-приложения «В хосписе»

## 1. Цель
- Проверить Android-приложение на соответствие требованиям функциональности посредством ручного и автоматизированного тестирования.
- Проверка UI/UX соответствия требованиям
- Автоматизация критичных сценариев

## 2. Описание приложения
Приложение «В хосписе» предназначено для работы с новостями хосписа и включает в себя: 
- информацию о новостях и функционал для работы с ними;
- тематические цитаты;
- информацию о приложении.

Основные модули:
- Авторизация и регистрация;
- Новости (список, создание, редактирование);
- Цитаты;
- Информация о приложении.

## 4. Подход к тестированию

### 4.1. Уровни тестирования
- **Unit-тесты**: проверка логики отдельных классов и методов (авторизация, проверка создания новости).
- **UI-тесты**: проверка пользовательского интерфейса и навигации между экранами. .
- **Ручное тестирование**: проверка UX, открытие ссылок на политику конфиденциальности и пользовательского соглашения.

### 4.2. Инструменты автоматизации
- **UI-тесты**: Espresso;
- **Инструменты запуска**: Gradle через Android Studio;
- **Отчётность**: через Allure.

## 6. Критерии успешности
- Покрытие unit-тестами ≥ 70% критически важной логики;
- Отсутствие критических багов (crash, потеря данных) при ручном тестировании;
- Проект собирается, тесты запускаются без дополнительных манипуляций;
- Проект работает на Android API 29;
- В проекте есть UI-тесты;
- Понятные названия тестов, методов и элементов;
- Нет явных изъянов в архитектуре, корректный code style.
